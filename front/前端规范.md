# 前端规范

本文档用于记录本项目 `front/` 目录的前端技术栈、工程约定与开发规范。

## 1. 技术栈

| 分类 | 技术 |
|---|---|
| 前端框架 | Vue 3，SFC，`script setup`，Composition API |
| 开发语言 | TypeScript 5.6 |
| 构建工具 | Vite 5 |
| 包管理器 | pnpm |
| 状态管理 | Pinia 2 |
| 路由 | Vue Router 4 |
| 样式 | Tailwind CSS 3 + PostCSS + Autoprefixer |
| HTTP 请求 | Axios |
| 国际化 | vue-i18n 9，支持 `en` / `zh` |
| 图表 | Chart.js + vue-chartjs |
| 单元测试 | Vitest + jsdom + @vue/test-utils |
| 覆盖率 | @vitest/coverage-v8 |
| 代码检查 | ESLint + eslint-plugin-vue + @typescript-eslint |
| 类型检查 | vue-tsc + vite-plugin-checker |
| 支付 SDK | Stripe、Airwallex |

常用业务库包括：

- `@vueuse/core`
- `@tanstack/vue-virtual`
- `marked`
- `dompurify`
- `xlsx`
- `file-saver`
- `qrcode`
- `driver.js`
- `vue-draggable-plus`

## 2. 项目结构

前端代码位于 `front/` 目录。

```txt
front/
├── index.html          # erp_web_app.py 托管的旧版 Web ERP 单页模板
├── src/
│   ├── api/              # API 请求封装
│   ├── components/       # 通用组件与业务组件
│   ├── composables/      # Composition API 复用逻辑
│   ├── constants/        # 常量定义
│   ├── i18n/             # 国际化配置与语言包
│   ├── router/           # 路由配置
│   ├── stores/           # Pinia 状态管理
│   ├── types/            # TypeScript 类型定义
│   ├── utils/            # 工具函数
│   ├── views/            # 页面级组件
│   ├── App.vue           # 根组件
│   ├── main.ts           # 应用入口
│   └── style.css         # 全局样式与 Tailwind 组件层
├── package.json
├── vite.config.ts
├── vitest.config.ts
├── tailwind.config.js
├── postcss.config.js
└── tsconfig.json
```

## 3. 开发命令

在 `front/` 目录下执行：

```bash
pnpm dev              # 启动开发服务器
pnpm build            # 类型检查并构建生产产物
pnpm preview          # 预览构建产物
pnpm lint             # 自动修复 ESLint 问题
pnpm lint:check       # 仅检查 ESLint 问题
pnpm typecheck        # TypeScript / Vue 类型检查
pnpm test             # 运行 Vitest
pnpm test:run         # 单次运行测试
pnpm test:coverage    # 运行测试并生成覆盖率
```

## 4. 构建与部署约定

- 构建工具使用 Vite。
- 构建输出目录为：`backend/internal/web/dist`。
- 前端构建产物由后端 Go 服务托管。
- 开发环境通过 Vite proxy 转发后端接口：
  - `/api`
  - `/v1`
  - `/setup`
- 开发后端地址通过 `VITE_DEV_PROXY_TARGET` 配置，默认值为 `http://localhost:8080`。
- 开发端口通过 `VITE_DEV_PORT` 配置，默认值为 `3000`。

## 5. 编码规范

### 5.1 Vue 组件

- 优先使用 Vue 3 Composition API。
- 单文件组件优先使用 `<script setup lang="ts">`。
- 页面级组件放在 `src/views/`。
- 可复用组件放在 `src/components/`。
- 复用逻辑应抽离到 `src/composables/`，命名使用 `useXxx.ts`。

### 5.2 TypeScript

- 项目开启严格类型检查，应尽量避免不必要的 `any`。
- 公共类型定义放在 `src/types/`。
- API 响应、表单数据、业务实体应定义明确类型。
- 路径别名使用 `@` 指向 `src`，例如：

```ts
import { useAppStore } from '@/stores/app'
```

### 5.3 状态管理

- 全局状态使用 Pinia。
- Store 文件放在 `src/stores/`。
- Store 应按业务域拆分，例如：
  - `auth`
  - `app`
  - `subscriptions`
  - `announcements`
  - `payment`
- 组件内部临时状态不应放入全局 Store。

### 5.4 路由

- 路由配置集中在 `src/router/`。
- 页面组件使用懒加载：

```ts
component: () => import('@/views/example/ExampleView.vue')
```

- 路由元信息通过 `meta` 维护，例如：
  - `requiresAuth`
  - `requiresAdmin`
  - `title`
  - `titleKey`
  - `descriptionKey`

### 5.5 API 请求

- HTTP 请求统一使用 `src/api/client.ts` 中封装的 Axios 实例。
- 业务接口按模块放在 `src/api/`。
- 不建议在组件中直接调用原始 `axios`。
- API Client 已统一处理：
  - `Authorization` token
  - refresh token
  - `Accept-Language`
  - 用户时区
  - 标准响应结构解包
  - 常见错误处理

### 5.6 国际化

- 国际化使用 `vue-i18n`。
- 当前支持语言：
  - `en`
  - `zh`
- 用户可见文案应优先使用 i18n key。
- 新增页面标题时，应同步维护路由 `titleKey`。

## 6. 样式规范

- 样式以 Tailwind CSS 为主。
- 全局基础样式与常用组件类定义在 `src/style.css`。
- 主题色、阴影、动画、字体等配置在 `tailwind.config.js`。
- 暗色模式使用 `class` 策略，通过 `html.dark` 控制。
- 项目未使用 Element Plus、Ant Design Vue 等大型 UI 组件库，主要采用 Tailwind + 自定义组件。

推荐写法：

```vue
<template>
  <button class="btn btn-primary">
    Submit
  </button>
</template>
```

## 7. 测试规范

- 单元测试使用 Vitest。
- Vue 组件测试使用 `@vue/test-utils`。
- 测试环境为 `jsdom`。
- 测试文件命名：
  - `*.test.ts`
  - `*.spec.ts`
- 测试文件应放在 `src/` 下对应模块附近，或 `src/__tests__/` 中。
- 覆盖率阈值目标：
  - statements: 80%
  - branches: 80%
  - functions: 80%
  - lines: 80%

## 8. 代码检查规范

- 提交前建议执行：

```bash
pnpm lint:check
pnpm typecheck
pnpm test:run
```

- 若需要自动修复格式或部分 ESLint 问题，可执行：

```bash
pnpm lint
```

## 9. 安全与内容处理

- Markdown / HTML 内容渲染时应使用 `dompurify` 做清理。
- 不应在组件中直接渲染未经处理的用户输入 HTML。
- 认证 token 相关逻辑应通过统一 API Client 与 Auth Store 处理。

## 10. 总结

本项目是一个基于 **Vue 3 + TypeScript + Vite + Pinia + Vue Router + Tailwind CSS** 的现代前端工程。整体架构偏轻量化，没有依赖大型 UI 组件库，主要通过 Tailwind 和自定义组件实现界面；业务侧包含用户面板、管理后台、支付、订阅、公告、OAuth、渠道监控等模块。
