店铺授权向导

Mercado Libre 授权向导

步骤 1：准备 Mercado Libre 开发者应用

App ID / Client ID 和 Client Secret 来自 Mercado Libre Developers，不是店铺登录账号。请进入 Mercado Libre Developers，创建或打开你的应用，在应用详情里复制。

授权配置检查清单

授权前先看这里，确认 App ID、Secret、Redirect URI、code 和 token 当前状态。

步骤 2：确认 Redirect URI

本模式与现有本地软件一致，默认使用 https://example.com/callback。请确保 Mercado Libre Developers 后台 Redirect URI 也是这个地址；如果另一个软件已经能授权，不要改后台配置。

步骤 3：打开授权链接

请使用已经登录 Mercado Libre 店铺主账号的浏览器打开授权链接。授权后可能跳转到 example.com 页面，页面打不开、403 或 404 都没关系，重点看地址栏是否有 code=xxx。

如果出现 403 / CloudFront / 参数缺失，请检查 App ID、Redirect URI 是否正确，并确保在 Mercado Libre 开发者后台保存了相同 Redirect URI。
步骤 4：粘贴授权回跳 URL 或 code

code 不是网址，不能直接打开。你可以粘贴完整回跳 URL，也可以只粘贴 TG-xxxx。code 是一次性的，有效时间短，使用后会失效。

整体设计说明

  1. 左侧显示平台授权卡片，快速查看 Mercado Libre、WB、Ozon 授权状态。
  2. Mercado Libre 授权分 4 步，不再直接堆叠 token 字段。
  3. 高级设置默认折叠，避免普通运营误操作 07D 调试按钮。
  4. Access Token、Refresh Token、Secret 等敏感信息默认脱敏显示。
  5. 支持默认浏览器、Chrome、Edge 打开授权链接；缺参数时不会打开错误链接。

字段解释

App ID / Client ID：Mercado Libre Developers 应用详情里的 ID。
Client Secret：Mercado Libre Developers 应用详情里的 Secret，只保存在本机。
Redirect URI：授权成功后 Mercado Libre 返回 ERP 的 callback 地址。
Access Token：授权成功后自动获取，用于调用平台 API。
Refresh Token：用于刷新 Access Token，自动保存。
Content API Token：WB 商品接口 Token。
Client ID / API Key：Ozon Seller 后台 API 信息。

常见错误

403 / CloudFront：授权链接参数错误、Redirect URI 未配置或平台拦截。
redirect_uri_mismatch：ERP 中的 Redirect URI 与开发者后台不一致。
invalid_client：App ID 或 Client Secret 错误。
invalid_grant：code 已失效、已使用或回调地址不一致。
token expired：Access Token 过期，需要刷新。
401：Token 无效或权限不足。
429：请求频率过高，被平台限流。

为什么默认使用 https://example.com/callback？

用户已有另一个本地软件使用同一个 Redirect URI 并能完成授权。为了不影响那个软件，本 ERP 默认也使用 https://example.com/callback。
授权后页面是否能打开不重要，重要的是从浏览器地址栏复制 code=TG-xxxx 回 ERP。
如果跳到 example.com 后页面打不开、403 或 404，不一定是失败。只要地址栏有 code，就复制整条地址。
ngrok / Cloudflare Tunnel 只保留在高级调试里，普通授权不需要。
