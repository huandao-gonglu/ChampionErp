# Step 28 - Phase 1 验收说明

## Phase 1 目标

集中打通 `Mercado Libre Mexico` 的真实运营闭环，而不是同时把多个平台做成空壳。

目标链路：

1. 商品采集进入母库
2. 批量认领到平台草稿
3. AI 文案 / 图片池推进状态机
4. 本地类目搜索与必填属性校验
5. 发布前预检
6. payload 预览与确认
7. 发布队列执行
8. 成功 / 失败回写与日志留痕

## 当前已达成

### 1. 底层数据与状态机

- 已从 `product.json` 过渡到 SQLite 主存储
- 商品、平台草稿、媒体、类目缓存、授权、日志已进入可持续扩展的表结构
- 草稿状态已能在页面里推进：
  - `collected`
  - `claimed`
  - `copy_ready`
  - `images_ready`
  - `ready_to_publish`
  - `published`

### 2. 发布前闭环

- 支持本地类目搜索
- 选择类目后自动加载必填属性
- 缺失属性会字段级标红
- 预检通过后商品库会进入可发布队列
- 发布按钮已改成两段式：
  - 先预检 + 生成 payload 预览
  - 再确认加入队列

### 3. 发布后闭环

- 批量发布任务面板已可刷新状态
- 失败原因会转成大白话
- “去修复”已支持字段级跳转与高亮
- 任务终态会回写到：
  - 当前商品草稿
  - 商品库列表
  - `publish_logs.json`
- 重复刷新同一个 `job_id + platform` 不会重复记日志

### 4. 页面验收

- 发布页已去掉主界面上的大段裸 JSON
- 原始结构、队列原始状态、调试快照已收进折叠区
- 主界面优先展示运营动作、平台卡片、确认卡片、任务摘要、日志
